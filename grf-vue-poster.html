GRF / VUE

Your vibe coding workflow — use it every session

GRF
Generate → Review → Refine
How you work with the AI to build something — not just accept whatever it gives you.
01
G
Generate
Give the AI a clear, specific prompt. Describe what you want to build, what it should do, and any constraints. The better your prompt, the better the output — vague prompts produce vague code.
Is my prompt specific enough that someone else could understand exactly what I want?
→
02
R
Review
Read what the AI produced. Don't just run it — read it first. Does it match what you asked for? Can you identify the shapes (accumulate, filter, find-best)? Does each comment match what the code actually does?
Can I explain what each block does before I run it?
→
03
F
Refine
Test it. Fix it. Adjust the prompt and regenerate if needed. You own the final product — not the AI. If something is wrong, go back to G with a better prompt, not just "fix this."
Does this actually do what I designed it to do, or just what the AI guessed I wanted?
⟳
GRF feeds into VUE
Once you've refined your code, you can't just ship it — you need to be able to explain it. VUE is how you check that your understanding is real, not just assumed.
↓
VUE
Verified → Understood → Explainable
How you check that you actually understand the code before you annotate or present it.
04
V
Verified
The code runs and does what you intended. You've tested it with real inputs, including edge cases. Working code is the floor, not the ceiling — it has to actually function before anything else matters.
Have I tested this with more than one input, including a tricky one?
→
05
U
Understood
You know what each part does and why. You can name the shapes. Your comments describe what the code actually does, not just what you hoped it would do. No copy-pasting AI explanations — rewrite in your own words.
Can I rewrite this comment from scratch without looking at the AI's explanation?
→
06
E
Explainable
You can walk someone through your code out loud — James, a classmate, a parent at exhibition — and answer follow-up questions. If you can't explain it, you don't own it yet. Go back to U.
Could I explain this to someone who hasn't seen it before, and answer their questions?
