Level 1 Single action, no decisions
Health potion pickup
Player picks up a potion — health increases by a fixed amount.
hidden
Flowchart
Shape: accumulate (decrement variant)
Annotated code
# Player picks up a health potion
# — health increases by 25

player_health = 60
potion_heal = 25

player_health = player_health + potion_heal

print("Health is now:", player_health)
✅ Match. The flowchart says "Add 25 to health" — the code does exactly that: player_health = player_health + 25. Shape: accumulate (a variable updated by adding a fixed amount). Comment, flowchart, and code all agree.
Level 2 Decision branch — something's off
Enemy encounter — attack or flee
Player decides whether to fight or run based on their health.
⚠ Mismatch
Flowchart
Shape: state-check (if/else branch)
Annotated code
# Decides whether to attack or flee
# based on player health

player_health = 15

if player_health > 20:
    print("You attack the enemy!")
if player_health <= 20:
    print("You flee from the enemy!")
⚠️ Mismatch. The flowchart has one diamond with two exits — a proper if/else. The code uses two separate if statements. That means both conditions get checked every time, even when the first one fires. This isn't a visual error — it's a structural one. If player_health were exactly 21, both conditions would still be evaluated (the second one just wouldn't trigger). The correct fix: if … else, not if … if. The flowchart's two-exit diamond should always map to an if/else in the code.
Level 3 Loop — watch the stop condition
Wave survival — fight until defeated or escaped
Enemies keep coming until the player dies or reaches the exit.
⚠ Mismatch
Flowchart
Shape: while loop with compound stop condition
Annotated code
# Player survives waves until
# they die OR reach the exit

player_health = 50
distance_to_exit = 40
wave_damage = 10
move_per_wave = 8

while player_health > 0:
    player_health = player_health - wave_damage
    distance_to_exit = distance_to_exit - move_per_wave

print("Game over!")
⚠️ Mismatch — two problems.

1. Missing stop condition. The flowchart's diamond has two conditions: alive AND not escaped. The code only checks player_health > 0. Even if distance_to_exit reaches zero (player escaped!), the loop keeps running. Escape is never detected.

2. Missing outcome message. The flowchart shows a "print outcome" box that says died or escaped. The code just prints "Game over!" regardless — it can't tell the difference because it never tracked the escape condition.

Fix: while player_health > 0 and distance_to_exit > 0: — then check which condition ended the loop before printing.
