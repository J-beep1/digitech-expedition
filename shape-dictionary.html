Core shapes — loops & collections
∑
Accumulate
Builds up a running total
Does X when you need to add things up across a whole list — totals, sums, counts.
Tell: variable updates every loop pass for +=
Example — total score
total = 0
for score in scores:
    total = total + score
↑
Find-best
Hunts for the highest (or lowest) value
Does X when you need the max or min from a list — top score, weakest enemy, cheapest item.
Tell: update only happens inside an if for if >
Example — strongest weapon
best = weapons[0]
for w in weapons:
    if w > best:
        best = w
⊂
Filter
Keeps some items, discards others
Does X when you want a subset of a list — passed students, alive enemies, items above a threshold.
Tell: .append() inside an if → builds new list for if .append()
Example — alive enemies only
alive = []
for enemy in enemies:
    if enemy["hp"] > 0:
        alive.append(enemy)
⇄
Transform
Converts every item into a new form
Does X when every item needs to be converted — raw scores → percentages, names → uppercase, pixels → coordinates.
Tell: same number of items in as out for dict[] =
Example — scores to percentages
grades = {}
for name in scores:
    grades[name] = scores[name] / max_score
Decision & loop-control shapes
?
State-check
Reacts when a threshold is crossed
Does X when something changes and the code needs to react — player dies, inventory fills, level up triggers.
Tell: if after an update, not inside a loop if else <= / >=
Example — health check after hit
player_health -= damage
if player_health <= 0:
    print("You died!")
else:
    print("Still fighting!")
↺
While-loop control
Repeats until something stops it
Does X when you don't know how many times the loop will run — game rounds, user retries, wave survival.
Tell: condition checked before every pass while and / or
Example — fight until dead or escaped
while hp > 0 and dist > 0:
    hp -= damage
    dist -= speed
⌕
Lookup
Fetches a value by name from a dictionary
Does X when you have named data — player stats, weapon damage, item descriptions — and need one specific value.
Tell: dict[key] — square brackets with a string or variable dict[key] .get()
Example — weapon damage lookup
weapons = {"sword": 15, "axe": 22}
item = "axe"
damage = weapons[item] # → 22
#
Count
Tallies how many items match a condition
Does X when you need a number, not a list — "how many students passed", "how many enemies remain". Not the same as filter!
Tell: count starts at 0, goes up by 1 inside an if count += 1
Example — count passing students
count = 0
for s in scores:
    if s > 15:
        count += 1 # a number, not a list
What do the brackets mean?
( )
Parentheses
Calling a function, or grouping maths. The values inside are the inputs the function needs.
print("hello")
len(scores)
range(10)
[ ]
Square brackets
Creating a list, OR looking up a specific item by position or key. Context tells you which.
[1, 2, 3] ← new list
scores[0] ← by position
player["hp"] ← by key
{ }
Curly braces
Creating a dictionary (named pairs) or a set. Key: value — the key is the label, the value is the data.
{"hp": 50, "xp": 120}
{"sword": 15, "axe": 22}
{} ← empty dict
Quick-reference — commands & what they do
Command What it does Outputs
len(x) Counts items in a list, string, or dict a number
sum(x) Adds all numbers in a list together a number
max(x) Returns the largest value in a list a single value
min(x) Returns the smallest value in a list a single value
print(x) Shows a value on screen screen output
range(n) Generates numbers 0, 1, 2 … n-1 (not n) sequence
list.append(x) Adds one item to the end of a list modifies list
list.sort() Sorts the list in place — smallest first modifies list
sorted(x) Returns a new sorted list, original unchanged new list
list[1:] Everything from index 1 onward (skips first item) new list
dict[key] Fetches the value stored under that key the value
dict.keys() All the keys (names) in a dictionary key list
dict.values() All the values (data) in a dictionary value list
for x in list: Runs the indented block once for each item varies
for k in dict: Loops over the keys of a dictionary varies
while condition: Keeps looping as long as condition is true varies
if … else: One branch fires — never both one path
if … if: Both conditions checked independently — could both fire 0, 1, or 2 paths
def name(x): Defines a reusable block of code function
return x Sends a value back out of a function the value
